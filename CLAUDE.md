# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Four-session interactive course teaching AI-powered portfolio management in Julia with Jupyter notebooks. Progresses from classical Markowitz optimization through adaptive rebalancing, HMM backtesting, to production operations with sentiment monitoring and escalation protocols. All data is synthetic, generated by JumpHMM.jl.

## Common Commands

### Run tests
```bash
cd code && julia --project=. -e 'using Pkg; Pkg.test()'
```

### Activate a session environment (e.g., session-1)
```bash
cd lectures/session-1
julia --project=. -e 'using Pkg; Pkg.instantiate()'
```
Each session's `Include.jl` auto-instantiates on first run if `Manifest.toml` is missing.

### Build API docs
```bash
cd code/docs
julia --project=. -e 'using Pkg; Pkg.instantiate()'
julia --project=. make.jl
```

## Architecture

### Shared package: `code/src/`
All computational logic lives in the `eCornellAIFinance` module (`code/`), organized into four files:
- **Types.jl** — 20 `mutable struct` definitions, all using `My*` naming convention, organized by session
- **Factory.jl** — `build()` constructors for all types (multiple dispatch)
- **Compute.jl** — ~45 exported functions: portfolio optimization, allocation (Cobb-Douglas/CES/log-linear), rebalancing engine, bandit solver, production simulation, sentiment/escalation
- **Files.jl** — JLD2/CSV I/O helpers; also exports dataset type aliases (`MyTrainingMarketDataSet`, etc.)

The module entry point (`eCornellAIFinance.jl`) defines JumpHMM aliases (`hmm_fit`, `hmm_tune`, `hmm_simulate`, `hmm_validate`) to avoid name collisions, and exports are grouped by session.

### Lecture notebooks: `lectures/session-{1..4}/`
Each session is self-contained with its own `Project.toml` (depends on `../../code`) and `Include.jl` (environment setup + package imports). Notebooks are split into Lecture (theory) and Example (hands-on) notebooks.

### Data: `code/src/data/`
Pre-trained models and datasets in JLD2 format (~270 MB). Includes S&P 500 OHLC data (2014-2024 training, 2025 testing), synthetic training datasets, and pre-trained JumpHMM/portfolio surrogate models.

### Tests: `code/test/runtests.jl`
Single comprehensive test file (~600 lines) using Julia `Test` framework. Organized by `@testset`: type construction, factory methods, then session-specific compute functions. Uses 5 synthetic test assets with known SIM parameters.

## Notebook Authoring Rules

Detailed formatting rules are in `formatting.md`. Key points:

- **Section pattern**: blockquote explanation (`> `) -> code cell -> short transition sentence. Never write long interpretive paragraphs after code.
- **Blockquote structure**: title line (`> __Title:__`) + empty `>` + content. Never have consecutive blockquotes without connective text between them.
- **Learning Objectives / Key Takeaways**: exactly 3 items, exactly 2 sentences each, format `* __[Title]:__ [description]`. No equations — use conceptual descriptions.
- **Language**: direct, simple, concise. Avoid adjectives. Active voice.
- **Math notation**: all norms must have explicit subscripts (`\|\cdot\|_{2}`), bold for vectors/matrices, non-bold for scalars. All LaTeX must render correctly.
- **Content integrity**: all generated text must be directly supported by notebook content. No unsupported claims.

## Model Restrictions

**Never use Claude Haiku for any task in this repository.** This includes notebook creation, review, editing, and compliance checking. Use Sonnet or Opus only. This is a permanent, non-negotiable rule documented in `formatting.md`.
